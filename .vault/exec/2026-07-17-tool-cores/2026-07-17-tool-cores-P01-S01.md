# Permit the native read built-ins Read, Grep, and Glob in autonomous mode for document-authoring roles so deterministic grounding is invocable without a local prompt (executor-core)

## Scope

- `src/vaultspec_a2a/graph/nodes/worker.py`

## Description

- Add the `NATIVE_READ_TOOL_NAMES` constant (`"Read"`, `"Grep"`, `"Glob"`) in `worker.py` naming the spawned CLI's own built-in read tools, which execute agent-side over the workspace fs with no MCP and no registration-scope surfacing gate.
- Add `_compose_native_read_tools(model, *, autonomous, role)`: for an autonomous document-authoring role only, union the read built-in names by exact name into the ACP model's `allowedTools`, preserving any existing allowlist entries (e.g. the bridged authoring tool names) and the advertised MCP servers unchanged; feature-detect the ACP allowlist surface via `with_mcp_servers` so mock/hosted models pass through unchanged.
- Wire the composition into `worker_node` after the authoring-tool and harness-server steps, so the read floor unions with, rather than replaces, whatever those set.
- Gate document-authoring scoping on the existing `is_document_authoring_role` contract (researcher, synthesist, adr-author, doc-reviewer); every other role (`role=None`) gets no read-built-in grant.

## Outcome

The deterministic grounding floor is live on the Claude/Z.ai ACP lane: an autonomous document role's `session/new` auto-permits exactly `Read`, `Grep`, and `Glob` (exact names, never a wildcard), invocable without a local prompt, while write/exec built-ins stay gated and the `.vault` deny remains write-only. Aligns with the R4 refinement of the edge-conformance ADR (exact-name allowlist, autonomous only, never human-in-loop). Codex is untouched here (no `with_mcp_servers` surface); its floor rides the read-only sandbox in Phase P04.

## Notes

New test module `test_worker_native_read_tools.py` drives a real ACP subprocess (the protocol simulator, no mocks) and asserts the recorded `session/new` allowlist: read built-ins present for an autonomous document role, unioned with the authoring names when a binding is attached, absent for a non-document role, and absent for a human-in-loop run. Direct unit tests on `_compose_native_read_tools` cover exactness, role scoping, autonomous gating, and dedup of a pre-existing entry. Full suite: 17 passed (new module plus the authoring-wiring and harness-wiring regression sets).
