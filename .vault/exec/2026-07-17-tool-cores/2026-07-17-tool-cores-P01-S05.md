# Prove live on the Claude lane that a document agent reads a named .vault ADR mid-turn and cites it, real run against the live engine with no mocks and zero .vault writes (executor-service)

## Scope

- `src/vaultspec_a2a/service_tests/`

## Description

- Add `test_tool_cores_floor_live.py` under `service_tests`, a live Claude-lane proof that a document agent reads a named `.vault` ADR mid-turn and cites it, with zero `.vault` writes.
- Extend the standing pw7 acceptance harness rather than fork a driver: reuse `_reachable_stack` for the infra gate and `AcceptanceHarness` (its `_mint`, `_run_start`, `_run_status`) for token-mint and run-start against the live `vaultspec-adr-research` preset.
- Observe non-materializing: consume the run's public v1 SSE progress stream at `GET /v1/runs/{run_id}/stream` and witness the mid-turn read and citation, then cancel the run before any review gate applies so no document materializes.
- Bind the assertions to the real event schema (`ServerEventType`, `ToolKind`, the `tool_call_start`/`tool_call_update` and `message_chunk` frames): a read tool-call of kind `read` whose `locations[].path` or `title` names the target ADR, and a message chunk citing that ADR by stem.
- Enforce zero writes with a before/after snapshot of the engine workspace `.vault` (path to mtime+size), asserting an empty created/modified/deleted delta.
- Select the target ADR from the live engine vault at run time (`_pick_named_adr`) so the prompt names a real document the agent can read.

## Outcome

The proof is code-complete and runnable, gated honestly. Grounding (rag-first): `vaultspec-rag search "live service test drives research_adr run against live engine" --type code` surfaced `service_tests/harness.py` and `service_tests/test_pw7_acceptance.py` as the standing live driver to extend; a follow-up `--type code` search on the run status/message surface led to the SSE frame schema (`api/schemas/events.py`) that carries the mid-turn `ToolCallStartEvent` (`locations[].path`) and `MessageChunkEvent` (`content`) evidence, since the `RunStatusResponse` projection carries no agent findings text.

The live evidence run is NOT captured in this session and is NOT reported as passing. Blockers, all external: the vaultspec engine is not discoverable from the authoring session (`resolve_engine()` returns None, `VAULTSPEC_ENGINE_SERVICE_JSON` unset) though the a2a gateway is up; no Claude credentials are present in-session; and Claude weekly usage is limit-gated until ~2026-07-20. The test skips with a runbook pointer under those conditions (the sanctioned pw7 infra-gate pattern) and is fail-loud when the stack is present.
