# Migrate the adapter dependency from the deprecated at-zed-industries claude-agent-acp to at-agentclientprotocol claude-agent-acp version 0.59.0 (executor-core)

## Scope

- `package.json`

## Description

- Repoint the sole dependency in `package.json` from the deprecated `@zed-industries/claude-agent-acp@0.23.1` to the renamed `@agentclientprotocol/claude-agent-acp@0.59.0`.
- Remove the stale `package-lock.json` and `node_modules`, then run a real `npm install` in the isolated worktree to regenerate the lockfile against the renamed package.

## Outcome

The migrated adapter installs cleanly (111 packages, 0 vulnerabilities) and its node entry `node_modules/@agentclientprotocol/claude-agent-acp/dist/index.js` self-reports version 0.59.0. The bin entry key is unchanged (`claude-agent-acp` to `dist/index.js`), so only the scope directory in the resolved entry path changes (`@zed-industries` to `@agentclientprotocol`). The vendored `@anthropic-ai/claude-agent-sdk` jumped from 0.2.83 to 0.3.207, past the 0.3.166 changelog entry that names the exact runtime-injection surfacing fix on the S20 registration-scope axis. The regenerated `package-lock.json` carries no `@zed-industries` residue and no deprecation marker.

## Notes

- The renamed package pulls a different transitive tree: it now depends on `@agentclientprotocol/sdk@1.2.1` (was the zed ACP SDK) alongside the bumped Anthropic agent SDK. Whether our ACP layer is compatible with the crossed 0.2.x to 0.3.x SDK surface is verified in P02.S08.
- The lockfile was fully regenerated (not merged), as instructed, so no `node_modules` junction is used in this worktree.
