# Regression-verify the ACP surface the provider layer targets against the migrated adapter: session-new shape, permission modes and allowedTools, mcpServers config key, capability flags, and server-initiated fs-RPC behavior (executor-core)

## Scope

- `src/vaultspec_a2a/providers/`

## Description

- Run the full non-service provider suite against the migrated adapter: 225 unit and middleware tests pass, unchanged from the pre-migration baseline.
- Run the existing live subprocess bridge test `test_real_agent_connects_to_authoring_bridge` against the migrated adapter: it passes, proving `initialize` (with `clientCapabilities.fs.readTextFile`), `session/new` with the `mcpServers` config key, and a real server-initiated MCP connection over HTTP all survive.
- Probe the migrated adapter's handshake result shapes with a real subprocess and confirm the exact fields our layer parses.
- Probe a real minimal prompt turn to observe the live turn-level surface (server-initiated RPCs and `session/update` notification types).
- Prove a real end-to-end turn through the production `AcpChatModel` on the Claude lane: streaming deltas and native tool-calling to a filesystem side effect.
- Add a durable, no-spend, real-subprocess regression test `test_acp_migration_surface.py` that guards the handshake surface across future adapter bumps, and register it in the tests `conftest` live-file set.

## Outcome

The ACP surface the provider layer targets is intact on `@agentclientprotocol/claude-agent-acp@0.59.0`; the 36-release jump (vendored SDK 0.2.83 to 0.3.207, crossing 0.2.x to 0.3.x) introduced no breaking change to the surface `_acp_session.py` and `_acp_protocol.py` depend on.

Confirmed live against the real migrated subprocess:

- `initialize` negotiates `protocolVersion: 1` — identical to the version our request hardcodes and to which our `fs/*` and `terminal/*` capability-gated RPC method names are keyed. The result carries `agentCapabilities` (a populated dict including `loadSession: true`, which our `session/load` branch checks) and `authMethods` (a list), the two fields `InitializeResult` parses. `agentInfo.version` self-reports `0.59.0`.
- `session/new` returns `sessionId` plus a `modes` block with `currentModeId: "default"` and an `availableModes` list carrying the full permission-mode set (`auto`, `default`, `acceptEdits`, `plan`, `dontAsk`, `bypassPermissions`) — the exact shape `SessionSetupResult` parses into `agent_modes`.
- The headless `_meta.claudeCode.options.allowedTools` auto-permit shape our `setup_session` emits is accepted without error.
- The `mcpServers` config key is accepted and the agent connects to an injected server at session setup (bridge test, HTTP transport).
- A real prompt turn streams `session/update` notifications of exactly the types our dispatch loop handles (`agent_message_chunk`, `tool_call`, `tool_call_update`, `available_commands_update`) and completes with `stopReason: "end_turn"`, which our `handle_client_response` detects to close the turn.
- The production `AcpChatModel.astream` path streams assistant deltas (returned the single word `pong`) and drives the native `Write` tool to a real filesystem side effect (`probe.txt` materialized with `pong`), proving both the streaming-chunk shape and the tool-calling loop survive end to end.

Regression coverage is codified as `test_acp_migration_surface.py` (service-marked, real subprocess, reaped before any prompt so no spend), added to the provider `conftest` live-file allowlist so it is not mis-marked as a pure unit test. Gate: ruff clean, ty clean, 225 non-service tests pass, the new surface test and the bridge test pass live.

## Notes

- Server-initiated fs-RPC observation: during the real read-a-file turn the migrated adapter issued ZERO `fs/read_text_file` RPCs — it satisfied the read with its agent-side native Read tool rather than delegating filesystem access to the client. This confirms the research's open question (whether the CLI issues fs RPCs when native tools are available) in the negative for the read path, and matches the ADR floor design where native Read/Grep/Glob execute agent-side. Our `fs/read_text_file` and `fs/write_text_file` handlers plus the `.vault` write-deny chokepoint remain present as defense-in-depth and are exercised at the handler-logic layer by the passing `test_acp_vault_deny` and `test_acp_security` suites; their protocol keying is unchanged because the adapter still negotiates `protocolVersion: 1`. The primary write-prevention path during native-tool turns is therefore the CLI's own permission mode and workspace sandbox, not our fs-RPC guard.
- Additive-only drift: the migrated adapter emits a new `usage_update` `session/update` type and a new `configOptions` key in the `session/new` result. Both are additive and harmlessly ignored by our dispatch loop and result parser (unknown types fall through the `elif` chain; unknown result keys are not read). No handler is added — this is a regression-verify step, not a feature step, and no breakage occurs. If a future step wants token-usage telemetry, `usage_update` is the seam.
- The live turns ran on the operator's logged-in Claude CLI credential (`~/.claude/.credentials.json`), reached via `CLAUDE_CODE_EXECUTABLE`; no token was written to disk. The two spend turns were minimal single-word prompts.
- Dedup and grounding: the new regression test reuses the existing live-subprocess pattern from `test_acp_authoring_bridge` (production `spawn_acp_process`, raw `initialize` + `session/new` framing, `service` marker, `conftest` live-file registration) rather than forking a parallel harness. The one shared primitive - the stdout frame reader both files need - was extracted to `_acp_frames.read_acp_frame` and the pre-existing bridge test was refactored onto it, so exactly one reader exists across the live provider tests. Discovery was done with grep and whole-file reads: the vaultspec-rag code and vault indexes for this worktree report zero indexed sections (the shared server on `:8766` had not indexed this checkout), so the rag-first path was unavailable and the sanctioned grep-plus-read fallback carried the locate.
