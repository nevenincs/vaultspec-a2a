# Security policy

## Supported versions

Only the current `main` branch receives security support. Older commits and
releases are not supported.

## Report a vulnerability

Don't open a public issue for a suspected vulnerability. Report it through
[GitHub private vulnerability reporting](https://github.com/nevenincs/vaultspec-a2a/security/advisories/new).

Include:

- The affected version or commit
- The potential impact
- Steps to reproduce the vulnerability
- A suggested mitigation, if known
- Your availability for follow-up questions

## What to expect

Maintainers assess each report and may ask you to validate findings or proposed
fixes. They coordinate disclosure based on the vulnerability and remediation
work. The project doesn't guarantee response or remediation times.

If you want public credit, state your preferred name or handle.

## Ask a security question

Use [GitHub Issues](https://github.com/nevenincs/vaultspec-a2a/issues) only for
non-sensitive questions that are safe to disclose publicly. Put vulnerability
details in a private report instead.
