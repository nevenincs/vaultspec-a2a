# Agents Gaps Research

**Date**: 2026-02-25
**Domain**: Agent Providers

## 1. Native Windows 11 Compatibility Constraints (Gap G1)

**Architectural Problem**: The core mandate dictates execution on Windows 11 PWSH without WSL. LLM Agent CLIs are overwhelmingly developed on macOS/Linux and assume POSIX compliance, specifically regarding TTY capabilities and process sandboxing.

**Inclusion/Exclusion Decision**:

- **Excluded for Native Windows**: OpenAI Codex CLI.
- **Rationale**: The official OpenAI Rust rewrite of the Codex CLI implements security via OS-level syscall sandboxing. It strictly relies on `sandbox-exec` (macOS) and `Landlock` (Linux). Because these APIs do not exist on the NT kernel, running the CLI on native Windows strips it of its primary security features and is officially unsupported.
- **Included with Strict Constraints**: Anthropic Claude Code CLI.
- **Rationale**: Claude Code uses standard Node.js (via Bun) cross-platform file execution. However, community bug reports confirm that executing it via `Git Bash` or `mintty` on Windows results in a fatal `Error: Raw mode is not supported on the current process.stdin`.

**Architectural Mandate**: The Orchestrator's Process Manager *must* explicitly invoke Claude Code using `powershell.exe -NoProfile -Command "claude"` to guarantee proper Windows Console API allocation for the TTY bridge.

## 2. Token Lifecycles and Subscription Bypass (Gap G2)

**Architectural Problem**: Using standard developer API keys incurs massive pay-as-you-go costs. We intend to wrap the CLIs to inherit the user's flat-rate consumer subscriptions (e.g., Claude Pro). If the cached session tokens expire mid-task, the orchestrator will crash due to unhandled `401 Unauthorized` errors.

**Implementation Reference**:

- **Excluded**: Interactive `/login` tokens. The OAuth tokens generated by standard interactive logins are aggressively rotated and typically expire within **8 to 12 hours**. They are entirely unsuitable for a headless orchestrator service.
- **Included**: Long-lived setup tokens.
- **Rationale**: The architecture must mandate that the user provisions the environment using the headless automation command.

```powershell
# Required provisioning step for the Orchestrator environment
claude setup-token
```

This generates a static OAuth token valid for **1 year**. The orchestrator will inject this token into the subprocess environment (`os.environ["CLAUDE_CODE_OAUTH_TOKEN"]`). This completely eliminates the need for the orchestrator to build complex logic to catch and handle `401` refresh race conditions.

## 3. GLM-5 API Compatibility (Gap G5)

**Architectural Problem**: Validating whether Zhipu's GLM-5 requires a custom schema translation layer to interoperate with MCP-standard JSON Schema tool definitions.

**Analysis & Rationale**:
Zhipu's "Coding Plan" API (`https://open.bigmodel.cn/api/paas/v4/`) is strictly **OpenAI SDK compatible**.

- It accepts the standard `[{"type": "function", "function": {"name": "...", "parameters": <JSON_SCHEMA>}}]` array payload.
- It supports parallel function calling natively.

**Decision**: No custom tool-schema translation layer is required. The orchestrator's base Python A2A Wrapper can seamlessly pass MCP JSON Schema definitions directly to the GLM-5 API via the standard `openai` python package by overriding the `base_url`.
